HTTP Console Web Components

Educational tools for visualizing HTTP request/response data

Overview

This project provides web components for displaying HTTP traffic in an educational format. Perfect for tutorials, documentation, debugging tools, or learning about HTTP.

Components

1. http-console

Displays a single HTTP request/response pair in wire format with syntax highlighting.

<http-console id="example"></http-console>

<script type="module">
  await customElements.whenDefined('http-console');

  document.getElementById('example').data = {
    request: {
      method: 'GET',
      url: '/api/users/123',
      httpVersion: 'HTTP/1.1',
      headers: {
        'Accept': 'application/json',
        'Authorization': 'Bearer token...'
      },
      body: null
    },
    response: {
      status: 200,
      statusText: 'OK',
      httpVersion: 'HTTP/1.1',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ id: 123, name: 'John' }, null, 2)
    }
  };
</script>

2. http-waterfall

Visualizes multiple HTTP requests with three view modes: List, Duration, and Waterfall.

<http-waterfall id="viewer" view="list"></http-waterfall>

<script type="module">
  await customElements.whenDefined('http-waterfall');

  document.getElementById('viewer').exchanges = [
    {
      request: { method: 'GET', url: '/api/data', ... },
      response: { status: 200, statusText: 'OK', ... },
      timing: { startTime: 0, endTime: 87, duration: 87 }
    },
    // ... more exchanges
  ];
</script>

Live Capture: Set capture="true" on http-waterfall to automatically capture all fetch() and XMLHttpRequest calls made by the page.

Features

  • Comprehensive syntax highlighting for JSON, HTML, CSS, and JavaScript
  • Dark mode support with auto-detection
  • Three visualization modes: List, Duration, and Waterfall
  • Live HTTP traffic capture
  • Expandable request/response details
  • Binary data handling
  • Responsive design
